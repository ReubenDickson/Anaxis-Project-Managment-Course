Project Charter Structure (Follow This Exactly)


Your Project Charter must include the following sections:


1. Project Title

Clear and descriptive.

Example:

CBT Application – Frontend Development Project


2. Project Background

Why this project exists.

Explain:

- The problem being solved

- Why a frontend is needed

- Who benefits from the system


3. Project Objectives

What success looks like.

Objectives must be:

- Clear

- Measurable

- Outcome-focused

Example:

i. To deliver a usable frontend MVP for the CBT system

ii. To enable students to take exams online

iii. To enable administrators manage tests


4. Project Scope (High-Level)

Define what is included and what is excluded.

In Scope (Examples):

- Frontend user interface

- Student exam workflows

- Basic admin dashboard

Out of Scope (Examples):

- Backend API development

- Payment systems

- Advanced analytics

Scope control is a project manager’s shield.


5. Key Stakeholders

Identify people or groups who:

- Influence the project

- Are affected by the project

- Can approve or block decisions

Examples:

i. Product owner

ii. End users (students)

iii. Administrators

iv. Development team



6. Assumptions

These are things you are assuming to be true without proof.

Examples:

- Backend APIs are stable

- Users have internet access

- Frontend will be web-based

Unstated assumptions cause project failure.



7. Constraints

Limitations you must work within.

Examples:

- Time (12-week course)

- Scope (frontend only)

- Resources (student team)


8. Success Criteria

How will you know the project succeeded?

Examples:

i. Frontend MVP delivered

ii. Core exam workflows functional

All work tracked in GitHub
