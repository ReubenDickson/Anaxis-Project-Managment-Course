Functional vs Non-Functional Requirements

Functional requirements define what a system MUST do (features/actions), while non-functional requirements define HOW it performs (qualities/constraints). Functional needs are mandatory actions (e.g., "log in"), whereas non-functional needs ensure performance, security, and usability (e.g., "log in within 2 seconds").

Functional Requirements describe what the system does.
Focus: Core functionality and business process.
Examples:

- The system shall allow students to start an exam
- The system shall submit exam answers
- The system shall display results after submission

Non-Functional Requirements describe how the system performs.
Focus: User experience, security and performance.
Examples:

- The system must be responsive
- The system must handle concurrent users
- The system must provide basic error feedback

Project managers must document both.
