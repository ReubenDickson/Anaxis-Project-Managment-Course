WEEK 4: Requirements Gathering & User Analysis

Week Objective

Many software projects fail not because teams cannot build, but because they build the wrong thing.

This week is about requirements discipline.

You will learn how to:

- Think from the user’s point of view
- Translate needs into clear requirements
- Control scope before development begins
- Prevent ambiguity from entering the project


By the end of this week, you will have formal, written requirements for the CBT frontend.

What You Will Achieve by the End of Week 4

By completing this week, you will:

- Define realistic user personas
- Map user journeys for the CBT system
- Identify functional and non-functional requirements
- Establish clear scope boundaries
- Create requirement documents that developers can trust

This week sets the foundation for backlog creation in Week 5.
