Week Objective

In real jobs, project managers almost never start from a blank slate.

Most software projects involve:

- Existing systems (also called legacy systems)

- Partially documented APIs or no documentation at all

- Decisions made before you arrived

- Technical constraints you did not choose

This week, you will learn how to take control of such a situation professionally.


By the end of this week, you will clearly understand:

i. What the CBT backend system can do

ii. What the frontend must support

iii. What is unclear, missing, or risky

iv. How to document system understanding as a project manager


What You Will Achieve by the End of Week 3

By completing this week, you will:

i. Analyze an existing backend system from a PM perspective

ii. Identify frontend functional areas

iii. Document known capabilities and limitations

iv. Raise clarification questions as formal project issues

v. Reduce uncertainty before requirements definition

This week is about clarity, not coding.
