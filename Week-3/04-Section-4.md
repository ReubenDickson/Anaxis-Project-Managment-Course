How to Analyze a Backend as a Project Manager

When reviewing an existing backend, focus on functionality, not implementation.

Look for the Following:

1. User Roles

Identify:

- Who uses the system?

- What permissions do they have?

Examples:

i. Student

ii. Instructor

iii. Administrator


2. Core Capabilities

Identify what the system can already do:

i. User registration and authentication

ii. Exam creation

iii. Exam submission

iv. Scoring or result storage


3. Data Flow

Understand:

i. How data enters the system

ii. What actions trigger backend logic

iii. What responses the frontend should expect


4. Limitations & Unknowns

Identify:

i. Missing documentation

ii. Ambiguous behavior

iii. Assumed workflows

These become project risks if ignored.
