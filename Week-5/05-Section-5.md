Product Backlog (What It Is)

The product backlog is a prioritized list of all work items required to deliver the product. Think of the Product Backlog as a "living" inventory. Unlike a static requirements document, it evolves as the team learns more about the product and the market. It provides a transparent view of what is planned next, ensuring everyone - from stakeholders to developers - is aligned on the current strategy.

A common way to visualize the backlog is an iceberg: items at the top are small, detailed, and ready for work, while items at the bottom are large, vague "epics" that will be broken down later.

Backlog Item Types
The backlog translates the product vision into actionable pieces. Here is how we typically categorize those items:
``` bash
Item Type                Focus                           Example
User Story               Direct value to a user.         "As a shopper, I want to save items to a wishlist."

Bug                      Repairing broken functionality. "The 'Submit' button is unresponsive on mobile."

Chore                    Technical/Infrastructure work.  "Upgrade the server to the latest security patch."

Spike                    Research and exploration.       "Investigate which payment API is best for our region."
```
The Product Owner (PO) is the gatekeeper of this list, constantly re-ordering it based on business value, risk, and dependencies.

The backlog is:

i. Continuously refined

ii. Owned by the product perspective

iii. The single source of future work
