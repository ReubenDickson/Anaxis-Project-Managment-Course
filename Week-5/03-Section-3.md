Acceptance Criteria (Non-Negotiable)

Acceptance criteria define when a user story is considered complete.

They:

- Remove ambiguity
- Protect quality
- Prevent “almost done” work

Acceptance criteria should be:

i. Clear

ii. Testable

iii. Written in simple language

Example:

User Story: As a student, I want to submit an exam so that my answers are recorded.

Acceptance Criteria:

i. Student can submit answers only once

ii. System confirms successful submission

iii. Submitted answers are saved correctly

iv. Student cannot edit answers after submission
