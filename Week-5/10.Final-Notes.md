Mapping GitHub To Agile

To use GitHub effectively for Agile development, we map GitHub's built-in features to standard Agile terminology. This allows your team to manage a "living" backlog and execute sprints directly where the code lives.

``` bash
Core Feature Mapping

Agile Concept               GitHub Feature                  Function in the Workflow

User Story/Task             Issue                           The fundamental unit of work. Contains the description and criteria.

Sprint/Iteration            Milestone                       A group of issues with a specific due date. Tracks progress toward a goal.

Product Backlog             Project (Table View)            A master list of all issues that haven't been assigned to a milestone yet.

Scrum/Kanban Board          Project (Board View)            A visual way to track issues as they move from "Todo" to "Done."

Story Points/Types          Labels                          Metadata used to categorize (bug, feature) or estimate effort (size: small).
```
By connecting these pieces, you create a system where moving an Issue to "Done" on your Project Board automatically updates the progress of your Milestone, providing real-time visibility to everyone on the team.
