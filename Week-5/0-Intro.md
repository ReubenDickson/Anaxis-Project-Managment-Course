WEEK 5: User Stories, Acceptance Criteria & Product Backlog Creation

Week Objective

Requirements describe what must be built.
User stories describe how work will be delivered.

This week, you will convert your documented requirements into actionable backlog items that can be planned, estimated, and executed using Agile/Scrum.

By the end of this week, your project must have:

i. A structured product backlog

ii. Well-written user stories

iii. Clear acceptance criteria

iv. Issues fully managed in GitHub

This is a critical transition point in the course.

What You Will Achieve by the End of Week 5

By completing this week, you will:

i. Understand how Agile teams break work down

ii. Write high-quality user stories

iii. Define acceptance criteria that prevent ambiguity

iv. Create a complete product backlog

v. Prepare the project for sprint planning in Week 6

Poor backlog quality leads to failed sprints. This week fixes that.
